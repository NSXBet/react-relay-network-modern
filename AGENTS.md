# AGENTS.md

Guidance for AI agents working in this repository.

## What this is

`@nsxbet/react-relay-network-modern` — a Network Layer for [Relay Modern](https://relay.dev/docs/guides/network-layer/).
It wraps `relay-runtime`'s `Network.create` with a composable middleware pipeline that can manipulate
requests/responses on the fly (auth headers, URL, retry, caching), batch multiple Relay requests into one
HTTP call, and support SSR. Works in browser, react-native, and Node. Relies on global `fetch`.

A TypeScript fork of [`react-relay-network-modern`](https://github.com/relay-tools/react-relay-network-modern)
by @nodkz. Published to npm as a library. The package ships ES modules (`import`) and CommonJS (`require`)
builds plus generated `.d.ts`/`.d.cts` declarations, all wired through the `exports` field. Output goes to
`dist/` (git-ignored, generated by `tsdown`).

## Layout

- `src/` — all source, **TypeScript** (`.ts`).
  - `index.ts` — public API surface; everything exported here (values **and** types) is the package contract.
  - `RelayNetworkLayer.ts` — the core class; builds the middleware chain and wires `Network.create`.
  - `fetchWithMiddleware.ts` — runs a request through the middleware chain.
  - `definition.ts` — shared types (`Middleware`, `MiddlewareNextFn`, `FetchOpts`, etc.).
  - `RelayRequest.ts` / `RelayRequestBatch.ts` / `RelayResponse.ts` — request/response wrappers.
  - `RRNLError.ts` / `createRequestError.ts` — error types.
  - `middlewares/` — one file per middleware (auth, batch, retry, cache, logger, perf, url, upload,
    error, progress, persistedQueries). Each default-exports a factory `(opts) => Middleware`.
  - `express-middleware/graphqlBatchHTTPWrapper.ts` — server-side batch endpoint helper.
  - `**/__tests__/` — Vitest tests colocated with the code they cover (+ `__snapshots__`, `__mocks__`).
- `examples/` — usage examples (not built/published).

## Middleware model

A `Middleware` is `(next: MiddlewareNextFn) => MiddlewareNextFn` where
`MiddlewareNextFn = (req) => Promise<RelayResponse>`. Middlewares compose around a request and can act
before/after `next`. There are two special variants (see `definition.ts`):
- **Raw middleware** (`isRawMiddleware: true`) — operates on the raw `FetchResponse` before parsing.
- **Sync middleware** (`{ execute }`) — short-circuits the network layer with a sync/observable result
  (used for cache hits). The constructor in `RelayNetworkLayer.ts` sorts middlewares into these buckets.

When adding a middleware: add the file under `src/middlewares/`, default-export a factory, export the
factory (and its `Opts` type) from `src/index.ts`, and add a colocated test. Document it in `README.md`.

## Conventions

- **TypeScript** for all source. The public type contract is re-exported from `src/index.ts`; `tsdown`
  generates the `.d.ts`/`.d.cts` from source — there is no hand-written `index.d.ts`.
- **oxlint** for linting; **tsc --noEmit** for typechecking. No ESLint/TSLint/Prettier/Flow/Babel.
- Runtime is pinned via `.tool-versions` + `mise.toml` (node + bun). **bun** is the package manager.
- `tsc --noEmit` typechecks the shipped source only; `__tests__`/`__mocks__` are excluded (run by Vitest).
- Node engine: `>=18`.
- **Conventional Commits** (semantic-release). Commit messages drive automatic versioning and publishing on
  `main`/`alpha`/`beta`. Use `fix:`, `feat:`, `chore:`, etc. — do not hand-edit the version.
- Tests are **Vitest**, run in `jsdom` with globals enabled, colocated in `__tests__/`. Snapshots are used;
  update intentionally (`vitest run -u`).

## Commands

```bash
bun install          # install
bun run coverage     # vitest with coverage (the test runner)
bun run test         # vitest run (no coverage)
bun run test:watch   # vitest --watch
bun run lint         # oxlint
bun run typecheck    # tsc --noEmit
bun run check        # lint + typecheck + coverage (full gate, matches CI)
bun run build        # build dist/ (ESM + CJS + .d.ts) via tsdown
```

CI (`.github/workflows/ci.yml`) installs the runtime via `mise`, then runs `lint`, `typecheck`,
`coverage`, and `build`. Run `bun run check` locally before pushing to mirror the gate.

## Agent notes

- Match existing style; don't reformat or refactor unrelated code.
- Changing anything exported from `src/index.ts` is a public API change — be mindful of semver impact via
  the commit type, and update the README accordingly.
- Don't edit `dist/` — it is generated by `bun run build`.
