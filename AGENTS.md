Guidance for AI agents working in this repository.

## What this is

`react-relay-network-modern` ŌĆö a Network Layer for [Relay Modern](https://relay.dev/docs/guides/network-layer/).
It wraps `relay-runtime`'s `Network.create` with a composable middleware pipeline that can manipulate
requests/responses on the fly (auth headers, URL, retry, caching), batch multiple Relay requests into one
HTTP call, and support SSR. Works in browser, react-native, and Node. Relies on global `fetch`.

Published to npm as a library. The package ships two builds: CommonJS (`lib/`) and ES modules (`es/`),
plus copied `.flow` type files and `.d.ts` declarations.

## Layout

- `src/` ŌĆö all source. JavaScript with **Flow** type annotations (`/* @flow */` headers).
  - `index.js` ŌĆö public API surface; everything exported here is part of the package contract.
  - `RelayNetworkLayer.js` ŌĆö the core class; builds the middleware chain and wires `Network.create`.
  - `fetchWithMiddleware.js` ŌĆö runs a request through the middleware chain.
  - `definition.js` ŌĆö shared Flow types (`Middleware`, `MiddlewareNextFn`, `FetchOpts`, etc.).
  - `RelayRequest.js` / `RelayRequestBatch.js` / `RelayResponse.js` ŌĆö request/response wrappers.
  - `RRNLError.js` / `createRequestError.js` ŌĆö error types.
  - `middlewares/` ŌĆö one file per middleware (auth, batch, retry, cache, logger, perf, url, upload,
    error, progress, persistedQueries, legacyBatch). Each default-exports a factory `(opts) => Middleware`.
  - `express-middleware/graphqlBatchHTTPWrapper.js` ŌĆö server-side batch endpoint helper.
  - `**/__tests__/` ŌĆö Jest tests colocated with the code they cover (+ `__snapshots__`, `__mocks__`).
- `examples/` ŌĆö usage examples (not built/published).
- `flow-typed/` ŌĆö third-party Flow type stubs.
- `src/index.d.ts` ŌĆö hand-written TypeScript declarations for the public API.

## Middleware model

A `Middleware` is `(next: MiddlewareNextFn) => MiddlewareNextFn` where
`MiddlewareNextFn = (req) => Promise<RelayResponse>`. Middlewares compose around a request and can act
before/after `next`. There are two special variants (see `definition.js`):
- **Raw middleware** (`isRawMiddleware: true`) ŌĆö operates on the raw `FetchResponse` before parsing.
- **Sync middleware** (`{ execute }`) ŌĆö short-circuits the network layer with a sync/observable result
  (used for cache hits). The constructor in `RelayNetworkLayer.js` sorts middlewares into these buckets.

When adding a middleware: add the file under `src/middlewares/`, default-export a factory, export it from
`src/index.js`, add Flow types, update `src/index.d.ts`, and add a colocated test. Document it in `README.md`.

## Conventions

- **Flow** for types in `.js` source. Keep `/* @flow */` headers. TS users are served by `src/index.d.ts`.
- **ESLint** (airbnb-base + prettier) and **Prettier** (`.prettierrc`) for `.js`; **TSLint** for `.d.ts`.
- Node engine: `>=14.21.3`.
- **Conventional Commits** (Commitizen + semantic-release). Commit messages drive automatic versioning and
  publishing on `master`/`alpha`/`beta`. Use `fix:`, `feat:`, `chore:`, etc. ŌĆö do not hand-edit the version.
- Tests are Jest, run in `jsdom`, colocated in `__tests__/`. Snapshots are used; update intentionally.

## Commands

```bash
yarn                 # install
yarn coverage        # jest with coverage (the test runner)
yarn watch           # jest --watch
yarn lint            # eslint + tslint + tsc --noEmit
yarn flow            # flow type check
yarn test            # coverage + lint + flow (full gate, matches CI)
yarn build           # build lib/ (CJS) and es/ (ESM) + .flow + .d.ts
```

CI (`.github/workflows/nodejs.yml`) runs `yarn coverage` on Node 14/16/18, plus `yarn lint` and `yarn flow`
on Node 14. Run `yarn test` locally before pushing to mirror the gate.
